# 泰迪杯 C 题项目使用指南

本指南面向参赛选手，介绍如何快速搭建环境、运行项目并提交竞赛结果。

---

## 一、环境搭建

### 1.1 Python 版本要求

本项目需要 **Python 3.12.13**，请确保使用正确的版本。

### 1.2 创建虚拟环境

**macOS / Linux:**
```bash
python -m venv .venv
source .venv/bin/activate
```

**Windows:**
```bash
python -m venv .venv
.venv\Scripts\activate
```

### 1.3 安装依赖

```bash
pip install -r requirements.txt
```

依赖包括：pandas、numpy、PyYAML、openpyxl、matplotlib、networkx、qstock、tushare 等。

### 1.4 Tushare Token 配置

项目需要 Tushare Token 来获取行情数据，有两种配置方式：

**方式一：环境变量（推荐）**
```bash
export TUSHARE_TOKEN=你的token
```

**方式二：配置文件**
编辑 `config/config.yaml`，在 `tushare` 部分填写：
```yaml
tushare:
  token: "你的token"
```

> 获取 Token：访问 [tushare.pro](https://tushare.pro) 免费注册获取。

### 核心模块简介

## 三、数据准备

### 3.1 事件数据（data/events/）

事件数据按分类存储在以下目录：
- `policy/` - 政策类事件
- `announcement/` - 公告类事件
- `industry/` - 行业类事件
- `macro/` - 宏观类事件

**JSON 格式要求：**
```json
[
  {
    "raw_id": "唯一标识符",
    "title": "事件标题",
    "content": "事件内容描述",
    "published_at": "2026-03-24 08:42:00",
    "source_name": "来源名称",
    "source_url": "来源链接"
  }
]
```

### 3.2 手工数据（data/manual/）

- `industry_relation_map.json` - 产业链映射关系
- `stock_universe.csv` - 股票池
- `sample_news.json` - 新闻样例
- `stock_financial_sample.json` - 财务数据样例
- `suspend_resume_sample.json` - 停牌复牌数据样例

### 3.3 行情数据

首次运行时会自动通过 Tushare/AKShare 采集行情数据，并缓存到 `data/raw/` 目录，后续运行直接使用缓存，避免重复调用 API。

## 四、运行流程

### 4.1 周度运行（最常用）

```bash
python main_weekly.py --asof 2026-04-20
```

**参数说明：**
- `--asof`：分析基准日期，格式为 `YYYY-MM-DD`，通常为周一日期
- `--config`：（可选）指定配置文件路径

**输出目录：** `outputs/weekly/2026-04-20/`

**生成文件：**
- `result.xlsx` - 竞赛提交文件
- `report.md` - 详细分析报告
- `final_picks.csv` - 最终选股明细
- `predictions.csv` - 预测结果
- `company_relations.csv` - 关联关系
- `kg_visual/` - 知识图谱可视化图片
- `event_study/` - 事件研究统计

### 4.2 历史回测

```bash
python main_backtest.py --start 2025-12-08 --end 2025-12-26
```

**参数说明：**
- `--start`：回测开始日期
- `--end`：回测结束日期

**输出目录：** `outputs/backtest/`

### 4.3 生成竞赛提交文件

如需单独生成 Excel 文件：

```bash
python generate_result_xlsx.py --input outputs/weekly/2026-04-20/final_picks.csv --output result.xlsx
```

## 五、竞赛提交流程

### 第一周提交

**时间窗口：** 4月20日 15:00 — 4月21日 9:00

**操作步骤：**
1. 在 4月20日 15:00 前准备好最新事件数据，放入 `data/events/` 对应目录
2. 运行周度流程：
   ```bash
   python main_weekly.py --asof 2026-04-20
   ```
3. 检查生成的 `outputs/weekly/2026-04-20/result.xlsx`
4. 上传 `result.xlsx` 到竞赛平台

### 第二周提交

**时间窗口：** 4月27日 15:00 — 4月28日 9:00

**操作步骤：**
1. 更新事件数据
2. 运行：
   ```bash
   python main_weekly.py --asof 2026-04-27
   ```
3. 检查并上传 `outputs/weekly/2026-04-27/result.xlsx`

## 七、常见问题

### Q1: Tushare API 调用频率限制

**问题：** 频繁调用可能触发限频。

**解决：** 
- 数据会自动缓存到 `data/raw/`，后续运行直接使用缓存
- 建议间隔运行，不要连续快速重试

### Q2: 某步骤运行失败

**解决：**
- 系统内置容错机制，会尝试降级继续运行
- 检查 `config/config.yaml` 配置是否正确
- 查看错误日志定位问题

### Q3: 如何添加新事件数据

**解决：**
1. 根据事件类型选择对应目录（`policy/`、`announcement/`、`industry/`、`macro/`）
2. 创建或编辑 JSON 文件，按格式添加事件对象
3. 重新运行 `main_weekly.py`

### Q4: 如何调整策略参数

**解决：**
编辑 `config/config.yaml` 中的 `strategy` 和 `scoring` 部分，调整以下参数：
- `max_positions`：最大持仓数量
- `single_position_max/min`：单只股票仓位上下限
- `positive_score_threshold`：正向评分阈值

## 八、配置参数说明

### 关键参数（config.yaml）

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `project.initial_capital` | 初始资金 | 100000 |
| `data.lookback_days` | 数据回溯天数 | 14 |
| `data.benchmark_code` | 基准指数代码 | 000300.SH |
| `strategy.max_positions` | 最大持仓数 | 3 |
| `strategy.single_position_max` | 单股最大仓位 | 0.5 |
| `strategy.single_position_min` | 单股最小仓位 | 0.2 |
| `strategy.min_listing_days` | 最小上市天数 | 60 |
| `strategy.min_avg_turnover_million` | 最小日均成交额（百万） | 80 |
| `strategy.positive_score_threshold` | 正向评分阈值 | 0.02 |
| `event_study.estimation_window_start/end` | 估计窗口起止 | -60 / -6 |
| `event_study.event_window_start/end` | 事件窗口起止 | -1 / 4 |
