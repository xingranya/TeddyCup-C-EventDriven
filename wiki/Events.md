# 事件识别与分类（Task 1）

## 识别流程

1. 从新闻数据按**时间窗口（36小时）+ 文本相似度（≥0.18 或 ≥2 共享关键词）**聚类
2. 每个聚类生成一个候选事件
3. 从聚类中选取代表性标题作为事件名

## 四维分类体系

| 维度 | 类别 |
|---|---|
| **subject_type**（驱动主体） | 政策类、公司类、行业类、宏观类、地缘类 |
| **duration_type**（持续周期） | 脉冲型、中期型、长尾型 |
| **predictability_type**（可预测性） | 突发型、预披露型 |
| **industry_type**（行业属性） | 军工、科技、新能源、低空经济、业绩预告 |

## 五大量化特征

| 特征 | 说明 | 权重/计算 |
|---|---|---|
| `sentiment_score` | 情绪得分（-1~1） | 正/负面词命中比 |
| `heat_score` | 热度得分 | 聚类条数×0.18 + 来源权重×0.55 + 新鲜度 |
| `intensity_score` | 强度得分 | 强刺激词×0.12 + 官方来源+0.15 + 金额词+0.1 |
| `scope_score` | 影响范围得分 | 涉及公司数 + 行业广度 + 政策/地缘加成 |
| `confidence_score` | 置信度得分 | 综合以上四维 |

## 关键词体系

事件分类基于预定义关键词库（`task1_event_identify.py:EVENT_TAXONOMY`），覆盖军工、科技、新能源、低空经济等主题。
